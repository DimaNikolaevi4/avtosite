## 1. ОКРУЖЕНИЕ И ИНФРАСТРУКТУРА

### 1.1 Сервер для ИИ-модели (Qwen 2.5 7B)

| Задача | Статус | Примечание |
|---|---|---|
| 1.1.1 Выбрать сервер для Ollama (VPS / домашний ПК / Colab) | ❌ | Ключевое решение. ПК должен работать 24/7. VPS за 300-500 руб/мес — самый надежный вариант. |
| 1.1.2 Установить Ollama на сервер | ❌ | `curl -fsSL https://ollama.com/install.sh \| sh` (Linux) или скачать с сайта. |
| 1.1.3 Загрузить модель `qwen2.5:7b-instruct-q5_K_M` | ❌ | `ollama pull qwen2.5:7b-instruct-q5_K_M`. Размер ~5.4 ГБ. |
| 1.1.4 Протестировать запуск модели в командной строке | ❌ | `ollama run qwen2.5:7b-instruct-q5_K_M "Привет, как дела?"` |
| 1.1.5 Настроить Ollama как сервис (автозапуск) | ❌ | `systemctl enable ollama` (Linux) или аналог. |
| 1.1.6 Проверить доступность API Ollama | ❌ | `curl http://YOUR_SERVER_IP:11434/api/generate -d '{"model": "qwen2.5:7b-instruct-q5_K_M", "prompt":"test"}'` |
| 1.1.7 Настроить файрвол (открыть порт 11434 только для IP хостингов) | ❌ | **Важно!** Не открывать порт в интернет без ограничений. Использовать `ufw` или обратный прокси с авторизацией. |

### 1.2 Бесплатные хостинги (12 шт.)

| Задача | Статус | Примечание |
|---|---|---|
| 1.2.1 Выбрать 2-3 надежных бесплатных хостинга с PHP | ❌ | Рекомендации: InfinityFree, HelioHost, AwardSpace. У каждого создать по 4-6 аккаунтов. |
| 1.2.2 Зарегистрировать 12 отдельных аккаунтов | ❌ | Использовать таблицу для учёта логинов/паролей/доменов. |
| 1.2.3 Убедиться, что на всех хостингах PHP >= 7.4 | ❌ | Проверить `phpinfo()`. |
| 1.2.4 Убедиться, что `allow_url_fopen` или cURL включены | ❌ | Критично для `simplexml_load_file()` или cURL-запросов. |
| 1.2.5 Проверить ограничения на размер файлов и дисковое пространство | ❌ | Запас в 3-4 ГБ на каждом хосте более чем достаточен. |
| 1.2.6 Подготовить структуру каталогов на каждом хостинге | ❌ | Корень: `index.php`, `fetch_and_rewrite.php`, `render.php`, папки `news/`, `template.html`. |

## 2. РАЗРАБОТКА PHP-СКРИПТОВ

### 2.1 Основной скрипт-сборщик (`fetch_and_rewrite.php`)

| Задача | Статус | Примечание |
|---|---|---|
| 2.1.1 Реализовать защиту по секретному ключу (`$_GET['key']`) | ❌ | Проверка в начале скрипта, чтобы запретить запуск посторонними. |

| 2.1.8 Реализовать сохранение результата в файл `.html` в папке `news/` | ❌ | Имя файла: `time() . '.html'`. |
| 2.1.9 Добавить проверку на дубликаты (по заголовку или URL новости) | ❌ | Хранить список обработанных ссылок в файле `processed.txt`. |
| 2.1.10 Реализовать резервный режим (если ИИ недоступен) | ❌ | Публиковать оригинальный текст с пометкой «Без обработки». |
